---
title: 'Search Applications'
description: 'Search and filter applications with pagination. Returns a list of applications matching the specified criteria. Applications are automatically filtered to your organization.'
openapi: 'GET /applications'
---

<Note>
This endpoint requires API key authentication via the `x-api-key` header. All results are automatically filtered to applications belonging to your organization.
</Note>

## Query Parameters

- `refId` (optional): Reference ID to filter applications for a specific customer within your organization
- `customer_id` (optional): Filter by customer ID
- `application_status` (optional): Filter by status (one of: not_started, in_progress, awaiting_partner, completed, submitted, approved, rejected)
- `limit` (optional): Results per page (1-100, default 50)
- `page_token` (optional): Token from previous response for pagination

## Example Request

```bash
curl -X GET \
  "https://api.incentives.leap.energy/alpha/applications?limit=25&application_status=submitted" \
  -H "x-api-key: leap_live_..." \
  -H "Content-Type: application/json"
```

## Example Request (Filtered by refId)

```bash
curl -X GET \
  "https://api.incentives.leap.energy/alpha/applications?refId=982734ihksjhfwoe8u&limit=50" \
  -H "x-api-key: leap_live_..." \
  -H "Content-Type: application/json"
```

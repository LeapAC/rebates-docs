## Setup

<Steps>
<Step title="Create your API Key">
  Generate API credentials from your partner dashboard for both sandbox and production environments.
</Step>

<Step title="Map devices to your system">
  Create a mapping between your product catalog and Leap's device IDs for eligibility checks.
</Step>

<Step title="Configure ownership of data requirements (Optional)">
  Define which data fields your system will provide vs. what customers will enter directly.
</Step>
</Steps>

## Get URL

Get a Connect URL to embed for eligible customers in your App.

<Info>
  **Options:** API, CSV Upload
</Info>

### Request

```json
POST /incentives
{
    "address": string,
    "building_type": enum,
    "device_id": uuid,
    "reference_id": string,
    "create_application": true
}
```

## Customer Fills the Application

Customer completes the application in Leap Connect. After completion, application_status is updated to `awaiting_partner` or `completed`.

<Note>
  The customer will provide required documentation and consent for data sharing during this step.
</Note>

## Upload Remaining Data Requirements

Provide information to complete the customer's application.

### Step 1: Search for Applications

Search for all applications in `awaiting_partner` status.

<Info>
  **Options:** CSV Upload, API
</Info>

### Request

```http
GET /applications
```

**Query Parameters:**
- `application_status`: Filter by status (options: `not_started`, `in_progress`, `awaiting_partner`, `completed`, `submitted`, `approved`, `rejected`)

### Response

```json
{
  "next_page_token": "242c3e24-43e6-4ad7-88b3-666cda21582b",
  "results": [
      {
          "application_id": uuid,
          "url": url,
          "application_status": string,
          "customer_devices": [
              {
                  "customer_device_id": uuid
                  // additional device fields
              }
          ],
          "customer": {
              "customer_id": uuid
              // additional customer fields
          },
          "required_attachments": {
              "invoice": true,
              "install_photo": false
          },
          "utility": {
              "auth_status": enum,
              "account_number": string,
              "meter_number": string,
              "account_status": enum
          },
          "contractor": {
              "contractor_name": string,
              "contractor_phone": string
              // additional contractor fields
          }
      }
  ]
}
```

### Step 2: Upload Partner Data

Upload the data requirements you provide for your customers to complete their applications.

<Tip>
  Check the `required_attachments` field to see which documents are needed for each application.
</Tip>
